**目标：** 在某矿区真实环境部署运行，替代微信群和电话调度

| 事项 | 周期 | 说明 |
|:----|:---:|:----|
| **服务器部署** | 1 周 | 采购/配置服务器，Docker 部署全套服务 |
| **网络环境** | 1 周 | 矿区 4G/5G 覆盖测试，离线降级方案验证 |
| **基础数据录入** | 3 天 | 导入真实司机/车辆/坑口数据 |
| **操作培训** | 3 天 | 调度员、司机、坑口管理员培训 |
| **并行试运行** | 2 周 | 新旧系统并行，收集反馈 |
| **正式切换** | — | 关停微信群调度，完全切换至系统 |

## 风险与缓解

| 风险 | 缓解措施 |
|:----|:--------|
| 矿山现场弱网/无网 | ✅ 离线调度已实现，SQLite 缓存 + 自动同步 |
| 司机手机操作不熟练 | 简化 UI、坑口管理员可代操作、提供培训 |
| 排队公平性争议 | 强制队列日志、人工调整必须记录原因 |
| 地磅型号不兼容 | 支持串口/蓝牙/手动三种模式 |
| 车牌识别准确率不足 | 端侧 MLKit + 置信度过滤，低置信度转人工 |
