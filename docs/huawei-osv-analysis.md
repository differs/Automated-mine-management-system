# 华为自己到底做不做调度系统？以及OSV条件

---

## 一、华为自己有调度产品吗？

### 华为的布局

| 产品/方案 | 说明 | 与我们的关系 |
|:---------|:----|:-----------|
| **矿山无人驾驶** | 无人矿卡的自动驾驶执行（车-网-云全栈） | ⚠️ **互补关系** |
| **矿鸿操作系统** | 设备互联互通的操作系统底座 | ✅ **合作基础** |
| **矿山AI大模型** | 100+AI算法（安全、巡检等） | ✅ **可对接** |
| **F5G/5G网络** | 矿山网络基础设施 | ✅ **基础设施** |
| **华为云** | 云计算、大数据平台 | ✅ **可部署在上面** |

### 关键判断：华为不做「运输调度管理」这个应用层

华为的定位非常清晰：

```
华为做：                     华为不做：
┌─────────────────────┐      ┌─────────────────────┐
│ 操作系统（矿鸿）      │      │                     │
│ 网络（5G/F5G/WiFi）  │      │   运输调度管理软件    │
│ 云计算（华为云）      │      │   运单管理/排队系统   │
│ AI平台（大模型）      │      │   司机App            │
│ 无人驾驶执行层       │      │   坑口管理            │
│ 摄像头/传感器        │      │   称重系统            │
│ 物联网平台           │      │   报表/看板           │
└─────────────────────┘      └─────────────────────┘
                              ↑ 这是我们的位置
```

**为什么华为不做？**
1. 运输调度是**行业应用软件**，不是基础设施
2. 华为的商业模式是卖ICT基础设施 + 操作系统授权
3. 这类细分应用需要贴近客户不断迭代，华为做不过来
4. 华为需要生态伙伴来填充应用层——这就是OSV存在的意义

**证据：**
- 华为官网的"矿山无人驾驶"页面是**自动驾驶执行**，不是**运输调度管理**
- 已认证的935款设备全部是硬件/驱动层，没有一个纯管理软件
- 华为官方定义OSV伙伴是"能提供联合解决方案的合作伙伴"

## 二、矿鸿OSV申请条件详解

### 申请条件

从官方认证指南来看，没有硬性的资质门槛（不需要注册资金、不需要行业资质），而是**综合评审**：

| 评审维度 | 具体内容 | 我们的情况 |
|:--------|:--------|:---------|
| **企业能力** | 是否有软件开发能力、团队配置 | ✅ Rust/Flutter全栈 |
| **鸿蒙经验** | 是否有OpenHarmony/HarmonyOS开发经验 | ⚠️ 需要补充 |
| **商业机会** | 是否有明确的商业计划和落地场景 | ✅ 有产品和客户 |
| **技术兼容性** | 产品能否与矿鸿技术架构对接 | ✅ Rust可交叉编译到ARM Linux |

### 认证测试内容

| 测试项 | 说明 | 是否需要做 |
|:------|:----|:---------|
| **硬件兼容性认证** | HAL层适配测试 | ❌ 我们是纯软件，不需要 |
| **系统基础兼容性** | 应用接口兼容性测试 | ✅ 可能需要 |
| **分布式兼容性** | 分布式服务能力测试 | ✅ 可能需要 |
| **系统增强兼容性** | 增强组件适配测试 | ⚠️ 视情况 |
| **矿鸿增强功能认证** | 矿鸿特色功能 | ⚠️ 视情况 |
| **MDTP能力认证** | 矿鸿设备协议 | ⚠️ 视情况 |

### 评分体系（满分100，60分及格）

评分标准不公开具体分数权重，但从官方描述可以推断：

```
企业资质与能力     ~30分  ✅ 我们有
鸿蒙技术经验       ~20分  ✅ **有AI/大模型经验，鸿蒙OpenHarmony基于Linux，技术栈可快速迁移**
商业计划与机会     ~30分  ✅ 有明确计划
产品成熟度         ~20分  ✅ 已有完整产品
```

### 时间与成本

| 项目 | 预估 |
|:----|:----|
| 申请周期 | 提交后1-2周有回复 |
| 认证周期 | 1年内完成即可 |
| 开发投入 | 2-3人月（适配矿鸿API） |
| 认证费用 | 免费（华为不收费） |
| 硬件成本 | 可能需要矿鸿认证设备做测试（可租用） |
