# 一期需求基线

## 1. 项目目标

构建一套矿山运输车辆调度系统，替换微信群、电话和 Excel 的人工调度方式，实现派单、排队、装车、称重和结算的数据闭环，并支持实时看板监控。

## 2. 一期业务边界

一期只解决运输调度主流程，不包含以下内容：

- 车牌识别自动入场
- 地磅串口自动采集
- 智能优化算法自动派单
- 历史合同和销售系统对接
- 复杂财务对账和付款能力

## 3. 角色定义

### 3.1 调度员

- 创建运单
- 给司机派单
- 查看各坑口排队情况
- 取消异常运单
- 处理司机未到场、临时插单等情况

### 3.2 司机

- 接收派单信息
- 查看当前运单状态
- 到达坑口后打卡入场
- 查看排队人数和预计等待时长
- 展示二维码给坑口管理员核验

### 3.3 坑口管理员

- 确认司机入场身份
- 管理坑口实时队列
- 开始装车、完成装车
- 发现插队或异常行为时上报

### 3.4 地磅操作员

- 录入毛重、皮重、净重
- 关联当前运单
- 提交异常重量说明

### 3.5 财务 / 运营

- 查看已完成趟次和吨位
- 查看运单状态和统计报表
- 导出结算数据

| 模块 | 功能 | 说明 |
|---|---|---|
| 用户与权限 | 登录、角色鉴权、司机绑定微信 | 后台和移动端登录方式可不同 |
| 司机管理 | 司机档案、车辆档案、状态管理 | 支持批量导入 |
| 坑口管理 | 坑口基础信息、启停状态 | 包含排队容量和位置 |
| 运单管理 | 创建、派单、取消、状态流转 | 一期核心对象 |
| 排队管理 | 到场登记、自动排号、叫号、离队 | 防插队的核心 |
| 装车管理 | 开始装车、结束装车 | 记录作业时长 |
| 称重管理 | 手工录入称重数据、异常校验 | 二期再接自动采集 |

## 5. 核心业务流程

### 5.1 标准流程

1. 调度员创建运单并选择坑口、司机
2. 系统将运单状态置为“已派车”
3. 司机在小程序查看任务并前往坑口
4. 司机到达后点击“我已到达”，系统确认到场并进入队列
5. 系统分配排队序号，坑口管理员按顺序叫号
6. 坑口管理员点击“开始装车”
7. 装车结束后司机前往地磅
8. 地磅读入或录入重量，运单完成

### 5.2 例外流程

- 司机超时未到场：调度员可取消或改派
- 司机到场但未入队：允许管理员手工补录，必须记录原因
- 装车后未称重离场：触发异常告警
- 司机放弃任务：运单取消并记录取消方和原因
- 同一司机不可同时执行多个进行中运单

## 6. 运单状态机

```text
待派车
  -> 已派车
  -> 已取消

已派车
  -> 已到场
  -> 已取消

已到场
  -> 排队中
  -> 已取消

排队中
  -> 装载中
  -> 已取消

装载中
  -> 已装载
  -> 已取消

已装载
  -> 称重中
  -> 已取消

称重中
  -> 已完成
  -> 已取消
```

## 7. 一期关键规则

- 一个坑口同一时刻只有一个活动队列
- 一个司机同一时刻最多有一个未完成运单
- 队列序号按“入队时间 + 人工调整记录”确定
- 所有手工干预都必须记录操作人、原因和时间
- 重量偏差超过阈值时必须进入异常记录
- 运单一旦完成，不允许直接修改关键重量字段，只能走更正流程

## 8. 指标口径

- 派单数：当天创建且状态不为取消的运单数
- 完成趟次：当天完成的运单数
- 平均排队时长：`开始装车时间 - 入队时间`
- 平均装车时长：`装车结束时间 - 装车开始时间`
- 坑口吞吐量：单位时间内净重累计
- 异常率：异常运单数 / 总运单数

## 9. 待确认事项

- 是否存在“司机接单确认”步骤
- 是否允许一个司机绑定多个车牌
- 皮重是否采用固定车辆档案值
- 称重是否一定有空车重和重车重两次记录
- 运费结算规则是否按坑口、车型、吨位分档
- 坑口是否存在班次概念和交接班需求
