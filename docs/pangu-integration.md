## 三、如何集成到我们的系统

### 方案A：API调用（最快）

```
我们的调度系统 → HTTP请求 → 盘古大模型API → 返回结果

适用场景：
  ✅ 智能派单优化
  ✅ 拥堵预测
  ✅ 异常检测
  ✅ 报表生成（自然语言）

成本：调用量计费（具体价格待查）
```

### 方案B：本地部署开源模型（可控）

```
我们的服务器上部署 openPangu 2.0 Flash（6B激活参数）
→ 完全本地运行 → 数据不出矿

适用场景：
  ✅ 数据安全要求高的客户
  ✅ 网络条件差的矿区
  ✅ 需要低延迟的实时调度

硬件要求：
  - openPangu 2.0 Flash（6B）：1张昇腾/GPU卡即可
  - openPangu 2.0 Pro（18B）：需要多卡集群
```

### 方案C：边缘部署（端侧）

```
在坑口边缘盒子上部署 Flash 版本（6B）
→ 现场实时AI推理
→ 离线也能用

适用场景：
  ✅ 弱网/无网矿区
  ✅ 需要毫秒级响应的场景
```

### 推荐方案：先API调用，成熟后本地部署

```
第1步：通过API对接盘古大模型
  → 实现智能调度优化
  → 不增加硬件成本

第2步：有客户需要数据安全
  → 本地部署 openPangu 2.0 Flash
  → 推荐用昇腾推理卡（华为官方支持）
```
