## 产品能做什么

### 核心流程全覆盖

```
调度员派单 → 司机接收 → 到场打卡 → 排队叫号
→ 装车确认 → 称重完单 → 数据看板
```

### 角色全覆盖

| 角色 | 使用方式 | 核心功能 |
|:----|:--------|:--------|
| 调度员 | Web 后台 | 派单、看板、异常处理 |
| 司机 | **Flutter App + 微信小程序** | 收任务、到场、看排队 |
| 坑口管理员 | **Flutter App + 微信小程序** | 叫号、装车、核验 |
| 地磅操作员 | 手机 App / 专业地磅终端 | 称重、完单 |
| 管理层 | Web 看板 | 实时产能、效率分析 |

---

## 核心技术能力

| 能力 | 说明 |
|:----|:----|
| **🧠 地形自适应算法** | 调度算法根据坑口布局、道路距离、路况等现场数据调参，非通用方案 |
| **📱 App + 小程序双模式** | Flutter App 完整功能 + 微信小程序免安装，司机零门槛 |
| **📡 实时调度** | WebSocket 毫秒级推送，调度变化实时可见 |
| **📴 离线可用** | 矿山没信号也能操作，网络恢复自动同步 |
| **📷 车牌识别** | 手机拍照即识别；可选配专用摄像头自动抓拍入场 |
| **⚖️ 地磅自动采集** | 串口直连专业地磅 + 蓝牙便携地磅 + 手机手动三模式 |
| **📍 电子围栏** | GPS 自动感知车辆到场，无需手动打卡 |
| **📊 实时看板** | 调度室大屏实时显示车、坑、单状态 |
| **🚛 无人驾驶预留** | 标准 API 接口，可对接无人矿卡系统 |
| **🐳 一键部署** | Docker Compose，30 分钟完成部署 |

## 技术栈

| 层 | 技术 |
|:---|:----|
| 后端 | Rust（axum + sqlx + tokio） |
| 数据库 | PostgreSQL + Redis |
| 移动端 | Flutter（Dart） |
| 前端 | Vue 3 + Vite |
| 实时通信 | WebSocket |
| 部署 | Docker + Nginx |
| 车牌识别 | Google MLKit（端侧，不上传图片） |
| 离线存储 | SQLite（本地缓存） |

---

## 与传统方案对比

| 维度 | 传统方案 | 本系统 |
|:----|:--------|:------|
| 移动端 | 专用硬件终端（贵） | **Flutter App + 微信小程序，手机即用** |
| 车牌识别 | 需布专用摄像头 | **手机拍照 + 可选配专业摄像头** |
| 地磅对接 | 需专业集成 | **串口专业地磅 + 蓝牙便携 + 手机三模式** |
| 部署方式 | 本地服务器 + 硬件集成 | **Docker 一键部署** |
| 离线能力 | 完全依赖网络 | ✅ **离线可用，有网自动同步** |
| 电子围栏 | 不支持 | ✅ **GPS 自动感知到场** |
| 合作模式 | 买断制，一次性付费 | **合作开发 + 订阅制，持续迭代** |
| 部署周期 | 数月 | **1-2 周** |
| 前期投入 | 50-100 万 | **门槛极低，按年订阅** |

## 适用场景

- 正在从人工调度向数字化转型的矿山
- 矿山短倒运输（采场 ↔ 堆场/破碎站）
- 多坑口、多车辆的高频调度
- 需要按趟次、吨位做日常管理的矿区
- 想先低成本试用，验证效果再决定是否全面上线的矿山
